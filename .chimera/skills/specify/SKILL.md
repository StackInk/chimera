# Chimera Skill: Specify

## When to trigger

Auto-triggered when feature enters `spec` phase.

## Behavior

You are in the **Specify** phase. Your job is to clarify requirements and produce a complete spec.md.

### Process

1. Ask clarifying questions one at a time (prefer multiple choice)
2. Identify edge cases and boundary conditions
3. Define acceptance criteria for each user story
4. Produce spec.md with: user stories (prioritized), functional requirements, success criteria
5. Self-review for ambiguity, placeholders, contradictions

### Constraints

- Do NOT write code or create files outside of spec documentation
- Do NOT skip to planning or implementation
- Each user story must be independently testable
- Mark unclear items as [NEEDS CLARIFICATION]

### Completion

When spec.md is complete and user approves, signal transition to `plan` phase.
